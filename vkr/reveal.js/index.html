Спецификация трассы

Мат. постановка задачи

Дано:
\(\mathcal{M} = (X, X_0, R)\) — система переходов (опр. 4.1).
\(\tau = (\sigma_1,\sigma_2,\dots,\sigma_m)\) — трасса программы (опр. 4.4).
\(C : \Sigma \times X \times X \to \{0,1\}\) — предикат согласованности шага (опр. 4.4).

Обозначим:
\(K(\mathcal{M},\tau)\) — множество допустимых длин согласованных с системой переходов префиксов трассы.

Найти:
Требуется найти такое значение \(k \in \mathbb{N}_0\), для которого показатель качества согласованности \(q(k)\) максимален при ограничениях:

\[ \left\{ \begin{aligned} &q(k) \to \max\limits_k,\\ &k \le m,\\ &k \in K(\mathcal{M},\tau). \end{aligned} \right. \]

Спецификация трассы

Тестирование системы верификации

  • Найдена ошибка в демонстрационном примере Two-Phase Commit: преждевременный Commit из-за повторных Prepared.
  • Найдены два несоответствия реализации Raft спецификации: currentTerm и self-RequestVote.
  • Расхождения локализованы по конкретным шагам трассы.
  • После исправления реализации трассы проходят проверку TLC.

Тестирование системы верификации

Достоинства и недостатки

Достоинства

  • сопоставление реализации с формальной TLA+ спецификацией;
  • локализация ошибки по конкретному шагу трассы;
  • не требуется логировать и определять все переменные модели;
  • повторное использование для разных реализаций и спецификаций.

Недостатки

  • точность зависит от качества инструментирования;
  • неполная трасса расширяет множество допустимых интерпретаций;
  • при недостатке данных возможно неоднозначное восстановление поведения.
